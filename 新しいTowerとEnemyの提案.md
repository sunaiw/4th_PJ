# 新規TowerおよびEnemy追加アイデアとゲームバランス提案

現在のゲーム仕様（通常タワー、ヒーラー、バリケードと、挙動の異なる各種エネミー、ウェーブごとのマップ拡張）を踏まえ、戦略の幅を広げつつゲームバランスに深みを与える新規のタワーとエネミーの追加アイデアを3案提案します。

---

## 案1：群衆処理とリスクの多様化（範囲効果 ＆ 分裂）

単体攻撃（追尾弾）への対抗策となるエネミーと、それを処理するためのタワーの組み合わせです。

### 📌 新規Tower: スプラッシュタワー (Splash Tower / 迫撃砲)
*   **配置コスト:** `4`
*   **最大HP:** `10.0`
*   **攻撃力:** `1.5`
*   **攻撃速度:** `0.5回/秒` (クールダウン2秒)
*   **射程範囲:** `3.5`
*   **アーマー:** `0%`
*   **仕様:** 弾がターゲットに命中した際、着弾点から一定半径内（例: 半径1.5）にいる全てのエネミーに対して等等に範囲ダメージを与える。
*   **役割:** ウェーブ後半の敵の数が劇的に増える場面（Wave 10以降の大量スポーン）において、通常の単体タワーのDPS不足を補う集団戦用タワー。

### 📌 新規Enemy: スライム (Slime / 分裂エネミー)
*   **移動速度:** `1.5`
*   **最大HP:** `8.0` (通常スケーリング適用)
*   **特徴:** タワー無視（Enemy2同様にすり抜け型）
*   **仕様:** HPが0になり倒された瞬間、その場に「小型スライム」（MaxHP 3.0 / 移動速度 2.5 / コアダメージ 1）を2体スポーンさせる。小型スライムは倒されても分裂しない。
*   **役割:** 単体攻撃タワーのターゲットを吸い込み（手数を浪費させる）、防衛ラインを突破する。スプラッシュタワーの範囲攻撃を当てることで、分裂後の小型個体もまとめて処理する戦略が求められる。

### 👉 バランス考案
分裂エネミーは「手数」の概念をプレイヤーに意識させます。現在の通常タワーだけでは撃退に時間がかかってコアまで押し込まれてしまうため、プレイヤーはコストを貯めてスプラッシュタワーを配置する必要性に迫られます。

---

## 案2：行動阻害（CC）とアウトレンジの脅威

敵を足止めする能力と、タワーの射程外から安全に攻撃してくる敵の組み合わせです。現在の「ヒーラー」の重要性をさらに高めます。

### 📌 新規Tower: フロストタワー (Frost Tower)
*   **配置コスト:** `3`
*   **最大HP:** `8.0`
*   **攻撃力:** `0.1` (微少ダメージ)
*   **攻撃速度:** `1.0回/秒`
*   **射程範囲:** `2.5`
*   **アーマー:** `20%`
*   **仕様:** このタワーの攻撃を受けたエネミーは、一定時間（例: 3秒間）「Frost状態」になり、移動速度および攻撃速度が **50%低下** する。（重ね掛け不可、持続時間の更新のみ）
*   **役割:** 自身単体では敵を倒せないが、重装（Enemy3）やボスの足を大幅に遅くし、他の通常タワーの攻撃機会を増やすサポートタワー。

### 📌 新規Enemy: スナイパー (Sniper)
*   **移動速度:** `1.5`
*   **最大HP:** `4.0` (低耐久)
*   **攻撃力:** `2.0` (高攻撃力)
*   **射程範囲:** `4.0` (通常タワーの射程3.0より長い)
*   **仕様:** タワーを最優先ターゲットとし、タワーの攻撃が届かないアウトレンジから攻撃を行う。
*   **役割:** バリケードやタワーを安全圏から削り取ってくるため、プレイヤーは「Range UP」の報酬を取ってタワーの射程を伸ばすか、ヒーラーを配置して回復し続けるかの対応を迫られる。

### 👉 バランス考案
純粋な火力によるゴリ押しだけではなく、「デバフによる時間稼ぎ」と「射程への対抗策」という新しい評価軸をプレイヤーの選択肢（報酬獲得や配置タイミング）にもたらします。

## 案3：リソース管理と持続力（エコノミー ＆ 高耐久）

タワーディフェンスの要である「コスト管理」に着目したタワーと、それを必要とするタフなエネミーです。

### 📌 新規Tower: ジェネレーター (Generator / コスト生成施設)
*   **配置コスト:** `5`
*   **最大HP:** `5.0`
*   **攻撃・回復機能:** なし
*   **仕様:** 設置しておくと、毎ウェーブのクリア時（報酬フェーズ移行時）に、プレイヤーの総コストに確定で **+1（または+2）の追加ボーナスコスト** を発生させる。（基本コストの上限には干渉せず、所持コスト自体を増やす）
*   **役割:** 早期に配置するほど終盤のコストに余裕ができるが、配置直後は防衛力が下がるため「いつ無理をしてジェネレーターを置くか」というリスク＆リターンのジレンマを発生させる。

### 📌 新規Enemy: リジェネレーター (Regenerator / 自動回復)
*   **移動速度:** `1.5`
*   **最大HP:** `12.0`
*   **仕様:** 最新の被弾から2秒間ダメージを受けなかった場合、毎秒自身のMaxHPの 15% 分のHPを急速回復し続ける。
*   **役割:** 配置の薄い箇所を通られると、道中で受けたダメージを回復されてしまい確殺が難しくなります。ルートを絞って（バリケード等で誘導して）「集中砲火」を浴びせるための戦略的配置が求められます。

# Gemini Flashへの実装依頼用テンプレート（プロンプト指示書）

これらのアイデアを実装としてGemini Flashに依頼する際、仕様漏れを防ぐための効果的なプロンプトの構成案です。
以下のテキストをコピーし、実装依頼時のベースとしてご活用ください。

```markdown
以下の仕様に基づいて、現在のタワーディフェンスのプロジェクトに新しい要素を追加してください。
既存のシステム（A*経路探索、タワー配置・売却ロジック、Rewardでのバフ適用など）と整合性を保つよう実装をお願いします。

【追加要素：スプラッシュタワー（迫撃砲）】
1. 新しいタワークラス（例: SplashTower.cs もしくは既存のTower継承元への機能追加）を作成・拡張してください。
2. 攻撃時に発射する弾（Projectile）に範囲ダメージロジック（Hit時に周囲Radius:1.5のエネミーをPhysics2D.OverlapCircle等で取得してダメージを与える）を実装してください。
3. プレハブのパラメータ初期値を設定し、UIの配置ボタンを追加してください。Costs: 4, HP: 10, Damage: 1.5, FireRate: 0.5, Range: 3.5。

【追加要素：分裂エネミー（スライム）】
1. 既存の Enemy クラスを継承、または分岐条件として SlimeEnemy クラスを作成してください。
2. Die() メソッド（またはHPが0になった際の処理）をオーバーライド/拡張し、親オブジェクトが消滅する直前に、現在座標に「小型スライム」のプレハブを2体インスタンス化する処理を追加してください。
3. 小型スライムもA*の経路上に即座に乗り、コアを目指すように移動ロジック（Waypointの引き継ぎ、または新規検索）を接続してください。
4. SpawnManager等に、該当ウェーブ（例: Wave 6以降10%の確率など）での出現抽選ロジックを追加してください。
```

> **Note:** 上記のテンプレートは「案1」を例にしています。案2のスロウ付与などを実装させる場合は、「状態異常（状態と継続時間の管理、Speedへの乗算処理）を追加して」等、処理の核となる部分を指示に盛り込むとGemini Flashはスムーズに実装を完了できます。
